Web Dev Trends 2018

Introduction

2017 telah menjadi tahun yang baik untuk pengembangan web - banyak teknologi baru seperti Web Components atau PWAs dapat digunakan di kebanyakan browser untuk pertama kalinya. Teknologi ini masih tergolong sangat baru.

Teknologi Apa yang akan bertahan? Apa yang lebih penting di tahun 2018? Mari kita lihat lebih dekat artikel ini. Satu kata yang sangat penting sebelum kita menyelam: Tren ini TIDAK dipesan oleh kepentingan!

Understand the Basics

Teknologi dasar dalam pengembangan website sangat penting karena ini akan tetap menjadi hal utama yang harus diketahui sebagai web developer pada tahun 2018! Kedengarannya sepele tapi karena kita memiliki banyak teknologi untuk pengembangan web, penting untuk memahami apa yang menjadi dasar dalam pembuatan website.

Tidak perlu menguasai semua teknologi ini - setiap web developer memiliki area pilihan, tapi kita perlu tahu bagaimana HTML, CSS dan JavaScript bekerja dan bagaimana mereka bekerja sama. Tidak ada jalan lain.

Dive into Node.js

Node.js mulai populer selama beberapa tahun terakhir ini dan akan terus meningkat. Node.js adalah serverside language yang hanya berupa JavaScript di server, jadi mengapa tidak mempelajari lebih dalam mengenai Node.js dan menjadi fullstack developer.Node.js menampilkan ekosistem yang dinamis dan aktif dengan banyak package third party yang dapat dipasang ke aplikasi.

Explore React, Angular and/ or Vue.js

Nama-nama tersebut merupakan framework yang memungkinkan Anda menciptakan user experience yang sangat menarik di aplikasi web karena semuanya dikontrol melalui JavaScript. Satu tren yang relatif baru yang mungkin akan berkembang cepat pada tahun 2018 adalah rendering pra-rendering atau server-side dari SPA. Itu berarti aplikasi yang dibuat masih dikendalikan oleh JavaScript dan berjalan di browser namun tetap harus melayani versi pra-render aplikasi pada muatan pertama. Ini meningkatkan SEO karena crawler melihat apa yang pengguna lihat, tanpa harus menunggu beberapa konten yang dimuat secara asinkron dan dapat mempercepat aplikasi.

Pengguna mungkin terus menyukai aplikasi web yang menarik, sehingga frontend framework seperti Angular, React atau Vue.js akan tetap penting di tahun 2018. Ketiga framework tersebut terlihat bagus saat ini. Developer dapat menemukan perbandingan antara ketiga framework tersebut, tapi akhirnya developer disarankan untuk menyelam ke banyak framework dan kemudian memilih framework favorit.

Web Components & Component Compilers

Template HTML memungkinkan Anda untuk menentukan kode HTML (menggunakan elemen ‘normal’ atau elemen khusus lainnya) di dalam elemen khusus Anda. Membangun komponen web / elemen kustom secara manual bisa menjadi sangat rumit. Karena itulah komponen-kompiler seperti StencilJS terlihat sangat menjanjikan.

Ide inti di balik proyek ini adalah memberi Anda sintaks yang bagus dan banyak pembantu - Stensil menggunakan fitur TypeScript, JSX dan Angular untuk membuat komponen web kustom. Kode Anda kemudian akan dikompilasi ke vanilla JavaScript, sehingga Anda dapat menggunakan komponen yang dibuat di halaman web manapun. Anda tidak perlu menggunakan framework tertentu atau semacamnya.

Kita antusias tentang komponen web karena menggunakannya memberi kita banyak kemungkinan untuk mengurangi ukuran bundle kita (Anda mungkin bisa membuat halaman web tanpa framework apapun!). Selain itu, elemen khusus ini sangat dapat digunakan kembali di seluruh proyek.

Use Static Page Generators

Developer harus membuat aplikasi web untuk perangkat yang lebih banyak dan berbeda yang dapat menjangkau semua jenis tempat dengan segala jenis konektivitas.

Generator Halaman Statis seperti Jekyll, Hugo atau proyek seperti Nuxt.js untuk Vue.js (ini terinspirasi oleh Next.js for React). Semua alat dan proyek ini memungkinkan kita melakukan satu hal: Buatlah halaman web yang kemudian kita buat di server lokal kita atau beberapa server build. Developer kemudian menyebarkan sekumpulan file statis - HTML, CSS dan JavaScript.

Keuntungan besar menggunakan alat tersebut adalah developer bisa mengirimkan halaman yang telah dirender sementara tanpa server. Hal tersebut dapat menghemat biaya. Satu hal yang perlu diingat: Jika developer membangun halaman yang menampilkan 10 dari 1000 halaman akan menjadi masalah dan disitulah kelemahannya.

Progressive Web Apps

Progressive Web Apps (PWA) adalah istilah yang merangkum beberapa teknologi yang memungkinkan developer membuat aplikasi web yang sangat mirip dengan aplikasi mobile.

Pengguna rata-rata memasang aplikasi mobile baru nol setiap bulan! Itu tidak terlalu banyak, bukan? Di sisi lain, orang mencari barang di Google setiap saat. Dan mereka menemukan: aplikasi web Anda. Bingo, itulah mengapa ide bagus untuk membawa yang terbaik dari kedua dunia - persis seperti yang dilakukan PWA.

Build Serverless Apps

Apa yg dimaksud dengan Serverless? Bagaimana seharusnya halaman web tanpa server bekerja? Jawabannya sederhana: tidak. Tapi istilah Serverless mengacu pada sesuatu yang berbeda. Serverless berarti kita tidak perlu khawatir mengelola dan memperbarui server, sehingga kapasitas server Anda meningkat drastis dan Anda tetap hanya membayar untuk apa yang Anda butuhkan.

Bagaimana cara kerjanya?

Anda hanya menulis kode Anda yang harus dijalankan sesuai permintaan , mengunggahnya ke layanan tanpa server (misalnya AWS Lambda), mungkin menambahkan beberapa layanan tambahan (misalnya, AWS API Gateway untuk membangun REST tanpa server API dengan mudah) dan Anda menerapkan aset statis Anda ke beberapa host statis seperti AWS S3.

Itu saja, aplikasi / kode Anda sekarang hanya berjalan bila diperlukan, yaitu saat pengguna berinteraksi dengannya, dan karena itu Anda tidak membayar waktu idle apapun. Selain itu, Anda tidak perlu khawatir mengkonfigurasi server Anda dengan benar karena tidak ada server.

Perlu juga dicatat bahwa sementara Amazon Web Services (AWS) adalah inovator yang kuat di bidang itu, sekarang kita mendapatkan lebih banyak lagi perusahaan yang menawarkan layanan serverless seperti Microsoft Azure maupun Google Cloud.

Jadi secara keseluruhan, pengembangan aplikasi serverless menjadi lebih menyenangkan, pengujian menjadi lebih mudah dan semakin banyak sumber daya untuk belajar bagaimana membangun aplikasi web serverless agar tersedia.

Artificial Intelligence and Bots

AI merupakan bidang yang sangat luas. Kita bisa menyelam ke Python, Machine Learning, Big Data, Deep Learning dan membangun alat AI sendiri. Jika Kita menguasai semua itu, kita akan memiliki keuntungan di masa mendatang.

Tools seperti AWS Lex, layanan yang memungkinkan anda membangun intelligent chat bot yang dapat Anda terapkan ke halaman Anda. Alat seperti ini membuat AI dapat diakses oleh kita semua dan memungkinkan Anda untuk berfokus pada ide dan logika bisnis anda, dan kita akan melihat banyak alat seperti ini muncul (dan menjadi lebih baik) di tahun 2018 - oleh karena itu ini adalah tren yang mungkin tidak ingin Anda lewatkan.

Summary

Apakah ini semua yang harus kamu pelajari?

Tidak, tidak sama sekali!

Masih banyak lagi yang bisa dipelajari dan dikuasai! Ada banyak teknologi terkait seperti DevOps yang mungkin juga perlu dipelajari. Tapi tren yang dibahas dalam artikel ini mungkin akan bertahan dan terus menjadi penting sepanjang tahun ini. Mungkin tidak pernah ada dunia yang lebih kompleks (dalam pengembangan web) tapi tidak pernah ada satu pun yang memiliki lebih banyak peluang. Belajar dan ambil peluang tersebut!
